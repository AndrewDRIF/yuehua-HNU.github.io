使用 Sqoop 进行数据交换

SQOOP = “ SQl to HadOOP”

前言

什么是大数据 ?

在大数据出现之前我们用 RDBMS 存储数据

  • RDBMS （Relational Database Management System）关系型数据库管理系统

这种数据库是最古老的数据库存储类型，只能存储结构化数据（structured data），

以行（row）和列（column）组成的表（table）存储数据，一组表组成一个库（database）

表与表之间的数据记录有关系（relationship）。现实世界中的各种实体以及实体之间的各种联系均用关系模型来表示。

关系型数据库，就是建立在关系模型基础上的数据库。

sql 就是关系型数据库的查询语句

但是问题来了, 在20年前 (本文写于2023年), 使用互联网的人很少, 所以互联网上流通的数据也很少,

但是随着互联网用户的增加, 在互联网上流通的信息量飞速增加, 达到 TB, PB, EB, ZB…

而且还伴随着图片, 视频, 反馈这种非结构化数据 (Unstructured data), 或 HTML, XML 这些半结构化数据 (Semi-structured data)

以上这些海量数据我们就称之为大数据

这就导致 RDBMS 的处境很尴尬, 所以大数据出现了

大数据的数据库如 HDFS, HBASE, Hive等, 可以轻松存储海量数据, 包括半结构化及非结构化数据

而且数据在增长, 内存的需求量也大大增加, 想要在 RDBMS 中存储可能需要经常删除数据

但是大数据有其自己的技术使数据进行压缩从而控制不断存储数据而从不 delete

为什么要用 Sqoop ?

我们虽然进入了大数据时代, 但是之前还有好多数据存储在 RDBMS 中, 我们如何对其进行数据迁移 ?

这时 Sqoop 出现了, 就像用勺子将哈根达斯挖到甜筒中一样

Sqoop 将 RDBMS 中的数据 import 到 HDFS 等大数据数据库中

而将 HDFS 等数据库中的数据 export 到 RDBMS 中

至于我们为什么要用到第二种 ?

有的小公司没有大数据技术, 但是还是想使用我们的数据, 怎么办 ?

我们就需要使用 Sqoop 来将我们存储在 HDFS 中的数据给到他们的 RDBMS 中
